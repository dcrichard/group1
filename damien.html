Responsive Web Design

What is responsive web design?

Not all users are the same

In a nutshell, responsive web design is about designing a webpage in such a way that it looks good for all users, altering the way it presents itself based the platform, screen size, or orientation in which it is being viewed.


A responsive design takes into consideration how to respond to a user's behavior and environment based on the type of device that a viewer may use. A page should look good if you are viewing it from a desktop, tablet, or mobile phone. This is done by determining how the images, text and navigation should appear based on the width of the viewport it is being displayed through.


Web developers can not keep up with the endless number of new resolutions and devices that their sites will be viewed on. To create a website version for each resolution and device would be very impractical. Responsive web design addresses this issue by using a combination of flexible grids and layouts, images, and the use of CSS and HTML to resize, hide, shrink, enlarge, or move page content around based on the size of the of the browser.


There are many different elements that can be modified so that the platform they are being displayed on and the way the user interacts with them is optimal regardless of platform, but a great experience can be achieved through many different solutions, it all comes down to how the developer would like the page to display in various resolutions, and using the tools and tricks available to him to ensure that the page does have the desired look and feel, regardless of where it is viewed.

What are the steps to creating a responsive design?

Consider these best practices

Take advantage of Device Features

Viewing a page on a mobile device the user has the capability to do unique tasks that they may not be able to perform on a computer such as sending a message, making a call, or opening an app. You should take advantage of these features and design your page in such a way that when it is viewed on a mobile device their experience is enhanced by leveraging these features.


The same goes for when your page is viewed on a large displayâ€¦ use the real-estate to your advantage to showcase your content.

Don't be afraid to Minimize

Mobile devices simply do not have the same amount of space to work with as a desktop computer. Because of this constraint you shuoldn't be afraid to remove some elements from your page when it is viewed on small devices to improve the users experience.


You could shrink all of your images and text, making your content condensed enough that everything you are trying to share is available on the page, but the users experience may be negatively impacted because they can't see what they are even reading, but doing so will likely discourage site visitors from reading or returning to your page.


When designing your page, consider your pages rendering on a small device as a site of its own. If content that you display on your desktop page isn't critical to conveying the message, leave it out or provide other ways for it to be obtained via smaller screens.

Prioritize your content

At times there will be content that your desktop page contains that you just can not remove from the page when it is viewed on a mobile device. When faced with the issue that all remaining content is absolutely pertinent to the visitor, you should consider prioritizing your content. Once prioritized you can display the content the user is most likely looking for at the top of the page, and the less important information towards the end of the page. You can also hide your lower priority content behind tappable areas so that the user has the ability to see the content if they wish.

Make your clickable areas consistant across the page

Because your users may be viewing your page content on mobile devices, they may not have the luxury of interacting with links and buttons using a precision pointing device such as a mouse, they will instead likely be using their fingers, which range in size considerably. To help with this you should consider having consistantly larger areas around interactive elements to improve the elements use.

Responsive Images

Not all devices need the same size images. A desktop may need an image thats 1400 pixels wide for clean look, but when displayed on a mobile device the image maximum available size may be 400 pixels wide. That means the size of the image that the mobile device is downloading would be 3 and a half times smaller than the desktop image, which is a considerable difference, especially when users may be using cellular data to download the page/image. Going with the largest size fits all approach, websites load much slower and consume more data. To optimize the display of images across devices you can use multiple versions of the same image that are tailored to the device they will be displayed on. According to statistics correlated from a Oct 2017 query on http archive, images still make up 54% of the weight of the web. Keeping this footprint small for mobile devices will go a long way towards increasing visitor satisfaction.

Responsive Text

Being able to read the content of your page is obviously a critical element to your pages success, but even more so when your page is being displayed on a mobile device. If a visitor can't read your page, then they won't be there long and likely will not return. You must ensure that your content is is legible regardless of where it is viewed. To do so you must consider both the font that you are using, as well as the size of the font. When your page is viewed on a smaller device, decrease the size of the font and make sure that there is a good balance between the headings and the body of the content.


According to a 2014 article in Smashing Magazine, when designing a responsive website, if small text on a mobile device requires the viewer to pinch and zoom, the responsive structure isn't really helping the user. Instead you should start with a comfortable font size, and measure to determine break points. From there you should adjust the font size in the various break points to evaluate readability and adjust if needed. A general rule of thumb is around 60-70 characters per line of body copy on a mobile device.

Use Scalable Vector Graphics

In addition to using multiple versions of the same image, you can also use Scalable Vector Graphics(SVGs) for things like icons. SVGs can scale to any size without losing their sharpness, which provides for a more polished look, regardless of the resolution of the site they are being viewed on.

Key elements used to create a responsive site

The viewport

As a page is rendered, the browser looks for a meta tag called the viewport. This tag provides the browser with instructions on how to control the page dimensions and scaling. To optimize the page for a wide range of devices the meta viewport tag must be added to the head of the html document. The following meta tag instructs the page to match the screenâ€™s width in device-independent pixels using the width=device-width statement, which allows the page to adjust content to match the screen size, while the initial-scale=1 statement instructs the browser to establish a 1:1 relationship between CSS pixels and device-independent pixels.

Images

When creating a website you want the images on that website to scale nicely to fit any browser size. You can control how an image scales by setting the width property on the image. When an image width is set to 100% the image will scale both up and down, but if the max-width property is set to 100%, the image will scale down if it has to, but it will not scale up to be larger than its original size. Setting these properties will allow your images to scale to different page sizes, but this alone does not handle all aspects of responsive images. To create responsive images, most web developers use the following three key attributes: srcset, sizes, and picture.


There is aResponsive Images Community Group that has many examples of how responsive images can work using the picture element and different media-queries that display how the images will look on different devices including: mobile devices, tablets, desktops, and monochrome/eInk.

Media Queries

Media-queries are a key element of css that enable webpage content to change to different screen sizes and resolutions based on the viewport. They are a fundamental componant of responsive web design, allowing designers to change the appearance of a webpage to match the device the page is being displayed on.
